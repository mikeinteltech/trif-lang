Getting started

Install TRIF via the platform-specific bootstrap scripts documented in README.md. The installer provides the native C++ compiler, ObjC shims for Apple platforms, runtime adapters, and the trif/trifc CLIs.

  1. Run trif create my-app --template web to scaffold a project with batteries-included defaults.
  2. Use trif run src/main.trif for an immediate execution path or trif compile src/main.trif -t javascript to emit browser-ready bundles.
  3. Switch the compiler target with trifc --target cpp|python|javascript while staying on the native pipeline.

Projects rely on the trif.json manifest. Configure aggressive error handling by setting "diagnostics": { "mode": "aggressive" } to fail fast on warnings, ambiguous types, or unsafe memory calls.

Hello server

A minimal HTTP service compiled directly to C++.

import std.http as http;
import std.io as io;

const server = http.createServer({ port: 8080 });

server.get("/", function(ctx) {
    return ctx.json({ status: "ok", at: http.now() });
});

export function main() {
    io.println("API ready on http://localhost:8080");
    server.listen();
}
Architecture overview

The native toolchain is implemented in modern C++ for maximum throughput. The compilation flow tokenises and parses TRIF source, produces an intermediate representation, and emits target-specific code generators. ObjC bridges expose Apple platform toolchains while keeping all critical logic in C++. Python and Node.js remain downstream compilation targets—produced from native emitters without executing any Python during compilation.

Lexer & parser
Implements streaming tokenisation and Pratt-style parsing with detailed diagnostics for aggressive error mode.
Code generation
Native C++ emitter produces optimised binaries; Python and JavaScript backends share the same IR lowering for parity.
Tooling surface
The trifc CLI orchestrates builds, incremental caching, diagnostics, and bundling through a unified interface.

Language guide

TRIF retains the readability of JavaScript while introducing deterministic semantics and aggressive error detection. Use let for mutable bindings, const for immutability, and prefer expression-oriented flow when authoring high-performance kernels.

Syntax essentials

  • let declares mutable bindings; const declares immutable values.
  • Functions use function name(args) { ... } with explicit return statements.
  • Primitive types cover numbers, strings, booleans, null, lists [...], and object literals { key: value }.
  • spawn dispatches asynchronous work via the runtime thread helpers.
  • Aggressive diagnostics flag shadowed bindings, implicit coercions, or unsafe pointer arithmetic.
let counter = 0;
const banner = "Ready";

function tick(step) {
    counter = counter + step;
    return counter;
}

Modules & exports

  • Import modules with import std.io as io; or import "./util.trif";.
  • ES module syntax is supported: import main, { helper as alias } from "./helpers";.
  • export function and export let expose named bindings; export default publishes a single value.
  • Re-export modules via export { name } from "./other"; or export * from "./shared";.
import std.io as io;
import logger, { formatTime } from "./time";

export function main() {
    io.println(formatTime(logger.now()));
}

Control flow & patterns

  • Pattern matching uses chained if/else or guard loops with while.
  • for (let item in collection) iterates arrays, lists, and runtime-provided iterables.
  • Use destructuring-style assignments with property reads: let host = config.host;.
  • Exceptions map to the runtime target: native std::runtime_error in C++, Python exceptions when targeting Python, and Error objects when targeting Node.js.

CLI reference

The trif executable bundles compilation, scaffolding, documentation, and the interactive REPL. The trifc binary exposes the low-level native compiler directly.

Command Description
trif compile <source> [-t target] Compile a TRIF file to C++, Python, or JavaScript.
  • -t/--target accepts cpp, python, or javascript.
  • -o/--output writes the compiled artifact to a custom path.
  • --encrypt <passphrase> applies simple XOR obfuscation for distribution builds.
  • --no-opt disables optimiser passes for easier debugging.
trif run <source> [-- args...] Compile a file to Python in-memory and execute immediately.
  • Arguments after -- are forwarded to main().
  • Use --no-opt to bypass optimisation in aggressive error mode.
trif create <name> [--template kind] Scaffold new workspaces with templates for lib, web, mobile, memory, or reverse.
  • Templates configure trif.json, entry points, and starter tests.
  • Web projects include Express-style HTTP servers; mobile templates emit PWA-ready manifests.
trif docs Launches this handbook from docs/index.html.
trif repl Interactive shell that compiles lines on the fly. Use :quit to exit.

Package manager (TrifPM)

TrifPM is a next-generation package manager prioritising speed, reproducibility, and security. Dependency resolution is written in C++ with an optimised SAT solver, yielding faster graphs than npm while guaranteeing deterministic lockfiles.

Command Description
trif package init [path] Create trif.json, src/main.trif, and an empty trif_pkg/ directory.
trif package install <name[@version]> Install dependencies from the active registry into trif_pkg/.
  • Offline registries are supported via bundled indexes.
  • Source archives are compiled to native IR before linking, ensuring consistent builds.
trif package install-local <path> Install from a local directory without registry calls.
trif package publish [path] Bundle the package and push it to ~/.trif/registry.
  • Creates a package.zip and metadata entry in index.json.
  • Sign releases with Ed25519 keys for tamper detection.
trif package list Display installed packages and versions inside trif_pkg/.
trif package use <url> Switch the active registry endpoint. Mirrors can be HTTP, file-based, or offline archives.

Standard library

Every module ships with the toolchain and is available without additional installation. Imports resolve immediately after running trif package install for third-party dependencies.

std.io

Streamlined file and console utilities.

API Description
println(value: Any) -> None Print a value to stdout immediately.
read_text(path: str) -> str Read a UTF-8 file into memory.
write_text(path: str, data: str) -> None Write a UTF-8 file, creating parents as needed.
read_json(path: str) -> Any Parse JSON from disk using read_text.
write_json(path: str, data: Any) -> None Pretty-print JSON to disk.
prompt(message: str) -> str Request console input from the user.

std.http

Express-style HTTP primitives for services and APIs.

API Description
class HttpServer(host: str = '0.0.0.0', port: int = 5000) Express-style HTTP server with a built-in router.
  • get/post/put/delete(path, handler) register route handlers receiving HttpContext.
  • listen() boots a background server thread; close() shuts it down gracefully.
class HttpContext Wrapper around the request with helpers for crafting responses.
  • text(value, status=200), html(markup), and json(payload) return HttpResponse.
  • send(body, status=200, content_type) streams arbitrary bytes.
createServer(config?) -> HttpServer Factory respecting host and port keys.
now() -> str Return the current timestamp (ISO 8601).
class HttpResponse Lightweight data class storing status, headers, and body.

std.memory

Advanced memory operations for high-performance workloads.

API Description
openBuffer(size: int) -> MemoryRegion Allocate a zero-filled buffer for manual reads/writes.
class MemoryRegion Typed reads and writes on a mutable buffer.
  • write32(offset, value) / read32(offset) access 32-bit unsigned integers.
  • write_bytes(offset, data) and read_bytes(offset, length) move arbitrary slices.
  • fill(value) initialises all bytes; search(pattern) finds sub-sequences.
  • to_bytes() exports the raw data as immutable bytes.
formatHex(value: int, width: int = 8) -> str Render values as zero-padded hexadecimal strings.

Additional modules

std.net

  • start_tcp_server(host, port, handler) launches a threaded TCP server.
  • send_tcp_message(host, port, message) connects and sends UTF-8 payloads.
  • broadcast_json(host, port, payload) serialises JSON and delivers via TCP.

std.data

  • load_csv(path) reads CSV rows into dictionaries.
  • save_csv(path, rows) persists dictionaries with headers.
  • filter_rows(rows, predicate) returns rows that match the predicate.
  • map_rows(rows, mapper) transforms rows using a mapper function.
  • to_json(data) pretty-prints JSON.

std.threading

  • spawn(fn) schedules work on the shared thread pool.
  • parallel_map(fn, items) maps concurrently across items.
  • sleep(seconds) blocks the current thread.

std.mobile

  • createApp(config?) prepares a mobile app container.
  • MobileApp.screen(name, builder) registers declarative screens.
  • MobileApp.export() emits a bundle consumed by mobile.build.

std.reverse

  • inspectExecutable(path) extracts section metadata from ELF/PE binaries.
  • hexdump(data, width) produces classic hex + ASCII dumps.
  • ExecutableInfo exposes path, format, and sections.

std.managers

  • TaskManager queues named callables and executes sequentially.
  • ResourceManager pairs manual resource acquisition/release callbacks.
  • LifecycleManager coordinates ordered lifecycle phases.
  • ResourcePool manages reusable connections with acquire/release.
  • PipelineManager composes ordered transformation steps.

Runtime API

The runtime adapters enable embedding TRIF output into host applications. The Python runtime powers execution across macOS, Linux, and Windows; the Node.js bridge leverages ECMAScript modules; the native runtime links directly with C++ targets. Diagnostics integrate with aggressive mode to surface cross-runtime issues.

API Description
Runtime.import_module(name: str) -> ModuleProxy Resolve stdlib modules, compiled artefacts, or on-demand .trif modules under trif_pkg/.
Runtime.execute_python(code: str, argv: list[str] | None = None) -> None Execute compiled Python code inside a sandboxed module and run main() or the default export.
Runtime.spawn(callable) -> None Launch a daemon thread executing callable (used by spawn statements).
Runtime.register_module_exports(name, exports, default) -> None Expose export tables to the module proxy system for interop.
Runtime.extract_export(proxy, name) -> Any Retrieve a named export from a module proxy.
Runtime.extract_default(proxy) -> Any Return the default export if supplied; otherwise return the wrapped module.
Runtime.prepare_project_environment(root: Path) -> None Add project roots and trif_pkg/ directories to the module search path.
ModuleProxy.get_export(name) -> Any Lookup helper for dynamic module inspection.

Tooling & workflows

High-performance computing

Use std.memory alongside the C++ backend for zero-copy buffer operations. The compiler emits intrinsics for SIMD and integrates with platform-specific allocators when available.

Web & mobile

Compile to JavaScript for browser and Node.js targets while reusing the same standard library contracts. Mobile builds export manifests for PWA shells or hybrid wrappers.

Reverse engineering

Combine std.reverse, std.memory, and manager utilities to orchestrate binary inspection pipelines.
